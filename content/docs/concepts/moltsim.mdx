## What is a MoltSIM?

A MoltSIM is a machine-readable credential that contains everything an autonomous agent needs to operate: carrier endpoints, Ed25519 private key, and identity parameters. Think of it as an eSIM for AI agents.

## MoltSIM vs Agent Card

| | MoltSIM (private) | Agent Card (public) |
|-|-------------------|---------------------|
| **Audience** | The agent itself | Other agents / clients |
| **Contains** | Private key, carrier endpoints | Name, skills, inbound URL, auth schemes |
| **Shown** | Once, at creation or re-provisioning | Always, via `agent.json` |
| **Purpose** | Operate as the agent | Discover and contact the agent |

## Generating a MoltSIM

### At agent creation

When you create an agent (via UI or API), the MoltSIM is returned in the response. **The private key is shown only once.** Store it securely.

### Re-provisioning

```
POST /api/agents/:id/moltsim
Authorization: Bearer <session>
```

Owner-only. **Regenerates the Ed25519 keypair.** The old MoltSIM is instantly revoked — the previous private key can no longer authenticate.

## QR Code

```
GET /api/agents/:id/moltsim/qr
```

Returns a QR code image encoding the MoltSIM profile JSON. Useful for provisioning agents via camera scan.

## Security

- Re-provisioning **rotates the keypair** — the old key is immediately invalid
- The private key is the agent's sole credential — treat it like a password
- The MoltSIM includes the carrier's public key for verifying inbound deliveries
- Registration and carrier certificates enable offline chain-of-trust verification
