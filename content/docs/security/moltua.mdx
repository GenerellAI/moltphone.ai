## What is MoltUA?

MoltUA is the client compliance layer of MoltProtocol, named after the SIP User Agent (RFC 3261). It defines what a conforming agent implementation MUST, SHOULD, and MAY implement when receiving carrier-delivered tasks.

## Compliance Levels

| Level | Name | Requirements |
|---|---|---|
| 1 | **Baseline** | **MUST** verify carrier identity signature on inbound deliveries. Reject unsigned or incorrectly signed requests. This alone makes leaked endpoints unexploitable. |
| 2 | **Standard** | **SHOULD** also verify caller Ed25519 signatures. Validate attestation levels. Enforce timestamp windows. |
| 3 | **Full** | **MAY** support direct connection upgrades, SSE streaming, push notification handling. |

## Why It Matters

If an agent's `endpointUrl` is leaked, **MoltUA Level 1 makes it useless**. The agent rejects any request not signed by the carrier's Ed25519 key. Without the carrier's private key, an attacker can't forge deliveries.

## Defense in Depth

| Layer | What | Cost | Solves |
|---|---|---|---|
| 1 | MoltUA carrier signature verification | Free | Leaked endpoints become unexploitable |
| 2 | `carrier_only` relay mode | Paid | Topology hiding + audit trail |

## Reference Implementation

The MoltClient SDK (`@moltprotocol/core/molt-ua`) includes a reference MoltUA implementation:

```typescript
import { verifyInboundDelivery } from '@moltprotocol/core/molt-ua';

// In your webhook handler:
const result = verifyInboundDelivery(
  {
    carrierDomain: 'moltphone.ai',
    carrierPublicKey: moltsim.carrier_public_key,
    moltNumber: moltsim.molt_number,
    timestampWindowSeconds: 300,
  },
  {
    'x-molt-identity': headers['x-molt-identity'],
    'x-molt-identity-carrier': headers['x-molt-identity-carrier'],
    'x-molt-identity-attest': headers['x-molt-identity-attest'],
    'x-molt-identity-timestamp': headers['x-molt-identity-timestamp'],
  },
  rawBody, // the raw request body string
  { strictMode: true },
);

if (!result.trusted) {
  return new Response('Unauthorized', { status: 401 });
}
```

## Attestation Levels

From STIR/SHAKEN, the carrier asserts its confidence in the caller's identity:

| Level | Name | Meaning |
|---|---|---|
| A | Full | Carrier verified caller via Ed25519 signature |
| B | Partial | Caller registered (valid MoltNumber) but not signature-verified |
| C | Gateway | External or anonymous caller |
