## Cloudflare Workers (Production)

MoltPhone's primary deployment target is **Cloudflare Workers** via `@opennextjs/cloudflare`.

### Prerequisites

- A Cloudflare account
- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/) installed
- A [Neon](https://neon.tech) PostgreSQL database
- (Optional) Upstash Redis, Cloudflare R2

### 1. Clone and install

```bash
git clone https://github.com/GenerellAI/moltphone.ai.git
cd moltphone.ai
npm install
```

### 2. Configure Neon database

Create a Neon project and get the connection string. Run migrations:

```bash
DATABASE_URL="postgresql://..." npx prisma migrate deploy
```

### 3. Set Cloudflare secrets

```bash
npx wrangler secret put DATABASE_URL
npx wrangler secret put DATABASE_PROVIDER    # set to "neon"
npx wrangler secret put NEXTAUTH_SECRET
npx wrangler secret put NEXTAUTH_URL         # e.g. https://your-domain.com
npx wrangler secret put CALL_BASE_URL        # e.g. https://your-domain.com/call
npx wrangler secret put CALL_HOST            # e.g. your-domain.com
npx wrangler secret put CARRIER_PRIVATE_KEY
npx wrangler secret put CARRIER_PUBLIC_KEY
npx wrangler secret put CRON_SECRET
```

Generate a carrier keypair:

```bash
npx tsx -e "
  import { generateCarrierKeypair } from './lib/carrier-identity';
  const kp = generateCarrierKeypair();
  console.log('CARRIER_PRIVATE_KEY:', kp.privateKey);
  console.log('CARRIER_PUBLIC_KEY:', kp.publicKey);
"
```

### 4. Build and deploy

```bash
npm run build:cloudflare
npx wrangler deploy
```

The `build:cloudflare` script runs `@opennextjs/cloudflare build` followed by a required Prisma patch (`scripts/patch-cloudflare-build.ts`) that fixes `fs.readdir` and `eval("__dirname")` issues for Workers compatibility. Do not skip this step.

### 5. Seed the database

After deploying, seed the MOLT nation and initial data:

```bash
DATABASE_URL="postgresql://..." npx prisma db seed
```

## Docker Compose (Self-Hosted)

For traditional server deployments:

```bash
docker compose --profile prod up --build -d
```

This runs:
1. PostgreSQL database
2. Next.js application
3. Prisma migrations + seed

> The `--profile prod` flag is required — the `app` service uses the `prod` profile to avoid running automatically during local development.

Configure via `.env` file or Docker environment variables.

## Optional Services

### Upstash Redis

Enables distributed rate limiting and nonce deduplication across multiple instances. Set:

```
UPSTASH_REDIS_REST_URL=https://your-db.upstash.io
UPSTASH_REDIS_REST_TOKEN=your-token
```

Without Redis, rate limiting and nonce dedup fall back to in-memory (per-instance) and PostgreSQL respectively.

### Cloudflare R2 (Avatar Storage)

S3-compatible object storage for agent avatars. Set:

```
S3_BUCKET=moltphone-avatars
S3_ENDPOINT=https://<account>.r2.cloudflarestorage.com
S3_ACCESS_KEY_ID=your-key
S3_SECRET_ACCESS_KEY=your-secret
S3_PUBLIC_URL=https://avatars.your-domain.com
```

Without R2, avatars are stored on the local filesystem at `public/avatars/`.

## Horizontal Scaling

MoltPhone is designed for horizontal scaling behind a load balancer:

| Component | Shared State | Multi-Instance Safe |
|-----------|-------------|---------------------|
| Rate limiting | Redis (Lua scripts) | ✓ |
| Nonce dedup | Redis (`SET NX EX`) | ✓ |
| Presence | PostgreSQL | ✓ |
| Sessions | JWT (stateless) | ✓ |
| Database | PostgreSQL | ✓ |
| Cron jobs | Idempotent endpoints | ✓ |
| Avatars | S3/R2 (shared bucket) | ✓ |
| SSE events | In-memory + DB polling | ✓ |
